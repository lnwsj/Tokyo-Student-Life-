# 📐 Design Principles — Tokyo Student Life

> 10 หลักการ — ทุก asset / scene / code ต้องเช็คกับ 10 ข้อนี้ก่อน ship

---

## 1. 🌡️ Mood > Detail

**ถ้าเลือกได้ระหว่าง:**
- (A) "ฉากละเอียดมาก แต่ดูแข็ง"
- (B) "ฉากเรียบง่าย แต่อบอุ่น"

→ **เลือก B เสมอ**

> เกมนี้ขายอารมณ์ ไม่ได้ขายความละเอียด
> Asset ที่ "น้อยแต่รู้สึกได้" ดีกว่า asset ที่ "เยอะแต่เฉยๆ"

---

## 2. 🌅 Time Tells Story

เวลาเปลี่ยน → แสงเปลี่ยน → mood เปลี่ยน

- 07:00 = แสงเช้านุ่ม อบอุ่น
- 12:00 = แสงแข็ง สดใส
- 17:30 = แสงทองยาว อบอุ่นสุด
- 19:00 = แสงผสม นีออนเริ่มเด่น
- 22:00 = แสงจากป้าย + ดาว

> ใช้ cycle นี้เป็น gameplay — ไม่ใช่แค่ visual

---

## 3. 🤫 Silence is OK

ไม่ต้องมี sound ทุกวินาที

- บางช่วงควร "เงียบ"
- ได้ยินแค่ wind + distant sound
- BGM หยุดได้ ไม่ผิด

> เกมนี้ "เงียบ" ได้ — เงียบแล้วรู้สึก "อยู่คนเดียว" ตามสไตล์ญี่ปุ่น

---

## 4. 🌐 Bilingual = Respect

TH + EN ไม่ใช่ "ลดทอนกัน" แต่ละภาษาแสดง context ต่างกัน

- **TH = อธิบาย** (narrative, hint, dialog)
- **EN = label** (HUD, menu, action)
- **JA = authentic** (in-scene signage)

> ห้ามแปลแบบ literal — ให้คิดว่า "ภาษาไหนเหมาะกับจุดนี้"

---

## 5. 👤 Character > Object

ทุก object ควรมี "เจ้าของ"

- รองเท้าเรียงหน้าบ้าน (มีคนถอด)
- แปรงสีฟันที่ใช้แล้ว (มีคนใช้)
- หนังสือเปิดค้างไว้ (มีคนอ่าน)
- แก้วกาแฟยังไม่ล้าง (มีคนดื่มเช้านี้)

> ไม่ใช่ prop เปล่า แต่เป็น "ร่องรอยของชีวิต"

---

## 6. 🚶 Walk, Don't Run

- Player เดินเป็นหลัก
- วิ่งได้แต่ไม่ควรใช้บ่อย
- เกมนี้คือการ "ใส่ใจ" ไม่ใช่ "เร่งรีบ"

> ความเร็ว default = walking pace ของคนจริง (~4 km/h)

---

## 7. 🔄 Loop is OK

ทำกิจวัตรซ้ำได้

- ไปโรงเรียน → กลับบ้าน → กินข้าว
- แต่ละ loop มีรายละเอียดเปลี่ยน:
  - คนผ่านต่างคน
  - แสงต่างกัน (เวลาต่างกัน)
  - เพลงต่างกัน
  - เมฆต่างกัน

> "ความเหมือน + ความต่างเล็กๆ" = ความรู้สึก "มีชีวิต"

---

## 8. 🎵 Sound = Half The Game

- BGM ต้องไม่ "กิน" หู
- ต้องเป็น background ที่ "รู้สึกได้แต่ไม่ตั้งใจฟัง"
- Ambient สำคัญกว่า BGM

> เกมนี้ปิดตา + เปิดหู = รู้ได้ว่าอยู่ที่ไหน (โรงเรียน ≠ ศาลเจ้า)

---

## 9. 🚫 No Hand-Holding

- ไม่มี tutorial pop-up
- ไม่มี "→ go here" marker
- ไม่มี quest log บังคับ
- ไม่บอก player ว่า "ต้องทำอะไร"

> ให้ผู้เล่นเดินเอง หาเอง ค้นพบเอง

---

## 10. 🏠 Honest Mundane

**อย่าทำให้ชีวิตประจำวัน "วิเศษ" เกินไป**

- เกมนี้คือชีวิตจริงของนักเรียนญี่ปุ่น
- ไม่ใช่แฟนตาซี
- ไม่มี "วันนี้วันพิเศษ"
- ทุกวันคือวันธรรมดา (และนั่นคือสิ่งสวยงาม)

> "ความสวยงามของวันธรรมดา" — Wabi-sabi

---

## การใช้ Principles

### เมื่อตัดสินใจ asset ใหม่
1. Mood > Detail ✓
2. Time Tells Story ✓ (รู้ไหมว่า asset นี้อยู่เวลาไหน?)
3. Sound = Half ✓ (asset นี้ส่งเสียงอะไร?)
4. Character > Object ✓ (ใครเป็นเจ้าของ?)
5. Honest Mundane ✓ (จริงไหม? ไม่วิเศษเกินไป?)

### เมื่อตัดสินใจ feature
1. No Hand-Holding ✓ (feature นี้บังคับผู้เล่นไหม?)
2. Loop is OK ✓ (ทำซ้ำได้ไหม?)
3. Bilingual = Respect ✓ (แสดง TH/EN ถูกที่?)

### เมื่อ optimize
1. Walk, Don't Run ✓ (performance สำคัญ — player เดินดูนาน)
2. Silence is OK ✓ (ไม่ต้อง process sound ทุก frame)

---

## ตัวอย่างการใช้

**❌ ผิด:** ตู้โทรศัพท์สาธารณะสีแดง มีไฟกระพริบ neon ที่หน้า
- → ไม่ Honest Mundane (ญี่ปุ่นไม่มีตู้แบบนี้แล้ว)
- → ไม่ Mood > Detail (เน้น visual ไม่ใช่ atmosphere)
- → ผิดย่าน (residential ไม่ใช่ Shibuya)

**✅ ถูก:** ตู้โทรศัพท์เก่าสีเงินซีด มีป้ายเล็กๆ ภาษาญี่ปุ่น ไม่มีไฟ
- → Honest Mundane
- → Character > Object (มีคนเคยใช้)
- → Time Tells Story (เห็นร่องรอยกาลเวลา)
