# 📐 Shape Language — Tokyo Student Life
## シェイプランゲージ

> กฎเรื่องรูปทรง — "Soft Organic > Geometric Hard"
> ทุก asset ในเกมต้องดู "นุ่ม" ไม่ "แข็ง" ไม่ "เป็นเหลี่ยม"

---

## Core Principle

> **"ถ้าเห็นแล้วรู้สึก 'math perfect' = ผิด"**

เกมนี้คือโลกของชีวิตจริง ไม่ใช่โลกของเรขาคณิต
ทุก shape ต้องมี subtle imperfection

---

## 1. Edge Treatment

### หลักการ
- ❌ ไม่มี sharp 90° corner (ยกเว้นจำเป็นจริงๆ)
- ❌ ไม่มี perfect straight line ยาว
- ❌ ไม่มี perfect cube / sphere / cylinder
- ✅ ทุก corner มี radius 2-5mm
- ✅ ทุก line มี subtle curve
- ✅ ทุก surface มี slight bump/dent

### ตัวอย่างเปรียบเทียบ

| Object | ❌ Bad | ✅ Good |
|--------|--------|---------|
| ตู้ขายของ | Box 90° ทุกมุม | Box + rounded corner 5mm + ขาตั้ง + ป้าย |
| อาคาร | Cube ตรงเป๊ะ | Cube + ระเบียง + ปล่อง + ป้าย + มุมโค้ง |
| เก้าอี้ | สี่เหลี่ยมแข็ง | สี่เหลี่ยม + โค้งที่พนัก + ขาโค้ง + เบาะนุ่ม |
| ต้นไม้ | ทรงกรวยเรขาคณิต | ทรงกลมไม่สมบูรณ์ + กิ่งไม่เท่ากัน |
| แก้ว | Cylinder ตรง | Cylinder + bottom หนา + rim โค้ง |

### กฎ Corner Radius
| Object | Radius |
|--------|--------|
| เครื่องใช้ไฟฟ้า | 3-5mm |
| เฟอร์นิเจอร์ | 5-10mm |
| อาคาร | 10-30mm (mild) |
| ยานพาหนะ | 5-15mm |
| ใบไม้ | n/a (natural curve) |
| เสื้อผ้า | n/a (natural drape) |

## 2. Form Language

### Organic Forms (Natural Objects)
ใช้สำหรับ: ต้นไม้, ใบไม้, ก้อนหิน, กองฟาง, เมฆ, น้ำ

**ลักษณะ:**
- ไม่สมมาตร
- ขรุขระ
- มีรายละเอียด (เส้น, รอย, ผิว)
- variation ในขนาด

**ตัวอย่าง:**
- ต้นไม้: แต่ละต้นต่างกัน (ไม่ใช่ copy-paste)
- ใบไม้: แต่ละใบเอียง ขนาด สี ต่างกัน
- ก้อนหิน: ไม่ใช่ sphere — มีเหลี่ยม มุม ผิว
- กองฟาง: ฟางแต่ละเส้นวางต่างกัน

### Geometric Forms (Man-Made Objects)
ใช้สำหรับ: ตึก, ประตู, หน้าต่าง, รั้ว, โต๊ะ

**ต้อง "modify" เสมอ:**
- เพิ่ม detail (ราว, มือจับ, ป้าย, ปล่อง)
- ใส่ wear (รอย, สีลอก, รา)
- Soften edge (chamfer, fillet)

**ตัวอย่าง:**
- ตึก: cube + ราวระเบียง + ปล่อง + ป้าย + คราบ
- ประตู: rectangle + มือจับ + รอยขีด + บานพับ
- รั้ว: ไม้กระดก + ช่องว่างไม่เท่า + เสาเอียงเล็กน้อย

### Pure Geometric = ❌
ห้ามใช้:
- Perfect cube (ไม่มี edge softening)
- Perfect sphere (ไม่มี bump)
- Perfect cylinder (ไม่มี variation)
- Perfect pyramid
- Perfect cone

## 3. Proportions

### Characters (Head : Body)
| Style | Ratio | Note |
|-------|-------|------|
| Realistic | 1 : 7 - 1 : 8 | ✅ เกมเรา |
| Anime | 1 : 5 - 1 : 6 | ❌ ผิด |
| Chibi | 1 : 3 - 1 : 4 | ❌ ผิด |
| Heroic | 1 : 7.5 - 1 : 8.5 | ✅ ผู้เล่น |

**ตัวอย่าง:**
- ตัวละครสูง 165cm → หัว ~22cm
- ตัวละครสูง 175cm → หัว ~23cm

### Buildings (W : H : D)
| Type | Ratio | Note |
|------|-------|------|
| House (1 story) | 1 : 0.8 : 1 | Wide, low |
| House (2 story) | 1 : 1.2 : 1 | Square |
| Apartment (3 story) | 1 : 1.5 : 1 | Vertical |
| School (3 story) | 1 : 1.4 : 1 | Long + tall |
| Shop (2 story) | 1 : 1.0 : 1 | Square |
| Shrine | 1 : 0.7 : 1 | Wide, low |

**❌ ห้าม:**
- 1 : 3 : 1 (ตึกสูงเกินไป เหมือนดาดฟ้า)
- 1 : 0.5 : 1 (แบนเกินไป เหมือนตู้คอนเทนเนอร์)
- 1 : 10 : 1 (เสาเข็ม ไม่ใช่ตึก)

### Trees (Trunk : Canopy)
| Type | Ratio | Note |
|------|-------|------|
| Sakura | 1 : 3 | แผ่กว้าง |
| Ginkgo | 1 : 3 - 1 : 4 | ทรงสูง |
| Bamboo | 1 : 8 - 1 : 12 | สูงมาก (กอ) |
| Pine | 1 : 2 - 1 : 3 | แผ่ |

**❌ ห้าม:**
- 1 : 1 (ทรงกลมบน trunk อ้วน)
- 1 : 10 (trunk บางเกิน เหมือนเสา)

### Furniture (Real Sizes)
| Object | Size (cm) | Note |
|--------|-----------|------|
| โต๊ะเรียน | 60 × 40 × 75 | standard |
| เก้าอี้เรียน | 40 × 40 × 80 | seat 45cm |
| เตียงญี่ปุ่น (ฟูก) | 100 × 200 × 15 | single |
| โต๊ะกินข้าว | 80 × 80 × 70 | low |
| ตู้เย็น | 60 × 65 × 180 | standard |
| อ่างล้างจาน | 50 × 40 × 15 | kitchen |

## 4. Avoid These Shapes (Anti-Shapes)

### ❌ 1. Perfect Geometric
- Perfect cube
- Perfect sphere
- Perfect cylinder
- Perfect pyramid
- ทุกอย่างที่ "math perfect"

### ❌ 2. Anime-Style
- Disproportionate head
- Sharp angular body
- ไม่มี anatomy detail
- Cel-shaded look
- แขน/ขายาวเกินจริง

### ❌ 3. Sci-Fi / Cyber
- Sharp angular mech
- Neon lines
- Holographic panels
- Floating geometry
- Hexagon/Octagon pattern

### ❌ 4. Horror / Gothic
- Spikes
- Sharp teeth
- Distorted proportions
- Dark voids
- Tentacles

### ❌ 5. Cartoon (Western)
- Squash and stretch
- Googly eyes
- Rubber hose limbs
- Pie-cut mouth
- 4 fingers

## 5. Use These Shapes

### ✅ 1. Soft Rounded
- ตู้ขายของ rounded corner
- เครื่องใช้ในครัว rounded
- เก้าอี้ rounded back
- โทรศัพท์ rounded edge
- รถยนต์ rounded body

### ✅ 2. Organic Natural
- ต้นไม้ irregular canopy
- ก้อนหินไม่เท่ากัน
- ใบไม้มีรอยหยัก
- คลื่นน้ำไม่สม่ำเสมอ
- เมฆ fluffy

### ✅ 3. Modified Geometric
- ตึก + ราวระเบียง
- ตึก + ปล่องไฟ
- ตึก + ป้าย
- ตึก + บันได
- ตึก + ตู้เมล์
- ตึก + กระถางต้นไม้

### ✅ 4. Slightly Imperfect
- กำแพงไม่ได้ระดับ 100%
- พื้นมีรอยแตก
- กระเบื้องไม่เท่ากัน
- หินเรียงไม่เป๊ะ
- กระดาษมีรอยพับ

### ✅ 5. Scaled Down
- ตึกใน bg มี detail น้อยกว่า fg
- ไม่ใช่ทุกอย่าง detail เท่ากัน
- Hierarchy: hero > standard > bg

## 6. ตัวอย่างการ Apply (Showcase)

### 🏠 บ้านญี่ปุ่น (Japanese House)

**❌ Bad:**
```
┌────────┐
│        │  ← flat box
│        │
└────────┘
```

**✅ Good:**
```
       ╱╲
      ╱  ╲      ← หลังคาเอียง
     ╱    ╲
┌───┴──────┴───┐
│  ╔══╗  ╔═╗  │  ← หน้าต่าง + ประตู (มีรายละเอียด)
│  ║▓▓║  ║▓║  │
│  ╚══╝  ╚═╝  │
├─══════════┤  ← ระเบียง
│  ╲      ╱  │
└─════════──┘
```

### 🥤 ตู้ขายของ (Vending Machine)

**❌ Bad:**
```
┌──────┐
│ ░░░░ │  ← flat box
│ ░░░░ │
│ ░░░░ │
└──────┘
```

**✅ Good:**
```
┌──────┐
│ ╔══╗ │  ← ป้ายด้านบน
├──────┤
│ ░▓▓░ │  ← สินค้า 1
│ ░▓▓░ │  ← สินค้า 2
│ ░▓▓░ │  ← สินค้า 3
├──────┤
│  ╔╗   │  ← ปุ่ม + ช่องหยอดเหรียญ
│  ╚╝   │
└──────┘
```

### 🌸 ต้นซากุระ (Sakura Tree)

**❌ Bad:**
```
   /\        ← ทรงกรวยเรขาคณิต
  /  \
 /    \
/______\
   ||
   ||      ← trunk ตรง
   ||
```

**✅ Good:**
```
  .  *  .    ← irregular canopy
 *  *   *    ← มีช่องว่าง มีรายละเอียด
.   *  .  *
  *   .  *
  |       |  ← trunk เอียงเล็กน้อย + มีกิ่ง
  |  /    |
  | /     |
```

### 🧑 คน (Character)

**❌ Bad (Anime):**
```
   ●          ← หัวใหญ่เกิน
  /│\         ← แขนผอม
   │
  / \         ← ขายาว
```

**✅ Good (Realistic):**
```
   ●          ← หัวขนาดพอดี (1:7)
  ┌─┐
  │ │         ← ลำตัว
  │ │  ╲╲
 ╱   ╲        ← แขนขาจริง
╱     ╲
```

## 7. การใช้ Shape Language ใน Workflow

### Step 1: Pre-Generation
ก่อน generate asset:
- [ ] กำหนดประเภท (organic / man-made)
- [ ] กำหนด corner radius
- [ ] กำหนด detail level
- [ ] กำหนด proportion ratio

### Step 2: Generation
ระหว่าง generate:
- ใช้ prompt ที่บอก "soft, rounded, organic"
- หลีกเลี่ยง prompt "sharp, angular, geometric"
- ใส่ detail (รอย, wear, imperfect)

### Step 3: Post-Generation
หลัง generate:
- [ ] มี sharp 90° corner ที่ไม่จำเป็นไหม
- [ ] มี perfect geometric ไหม
- [ ] proportion ถูกต้องไหม
- [ ] มี imperfection พอไหม
- [ ] มี detail เพียงพอไหม

### Step 4: In Scene
ใน scene:
- [ ] mix organic + modified geometric
- [ ] detail hierarchy (fg/bg)
- [ ] scale variety (ไม่ใช่ทุกอย่างเท่ากัน)
- [ ] no pure math-perfect objects
